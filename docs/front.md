# 주차별 프론트엔드 세부 개발 일정표

## 1주차 - 프로젝트 초기 세팅 + 채팅 UI 설계

**목표:** 프론트 기본 구조 완성 + 채팅 화면 UI 초안 제작

**세부 항목**

- 프로젝트 생성 및 폴더 구조 정리
- 공통 컬러, 폰트, 버튼, 입력창 스타일 정의
- 화면 라우팅 구조 설계
- 채팅 메인 화면 UI 제작
- 사용자 질문 입력창, 전송 버튼, 답변 출력 영역 구현
- 로딩 상태, 빈 화면 상태 UI 제작
- 백엔드 채팅 API 명세 확인 및 응답 구조 파악

**완료 기준**

- 앱/웹 기본 실행 확인
- 채팅 화면 UI 1차 완성
- 공통 컴포넌트 구조 정리 완료

—

## 2주차 - 채팅 기능 연동 + SSE 스트리밍 처리

**목표:** 채팅 API 연동 및 실시간 답변 출력 구현

**백엔드 연계 포인트:** 1주차 백엔드 채팅 API 완성 시점과 맞물림

**세부 항목**

- POST `/api/chat/ask` 연동
- user_id, session_id, question 요청 구조 맞춤
- SSE 스트리밍 응답을 화면에 실시간 출력
- 답변 생성 중 타이핑 효과 또는 단계별 렌더링 처리
- sources 필드 표시 UI 구현
- 오류 응답, 타임아웃, 서버 미응답 예외 처리

**완료 기준**

- 실제 질문 전송 후 답변 스트리밍 출력 확인
- 출처 정보 표시 확인
- 로딩/에러 UI 동작 확인

—

## 3주차 - 히스토리 화면 구현 + 조회/삭제 연동

**목표:** 대화 기록 목록/상세/삭제 기능 구현

**백엔드 연계 포인트:** 백엔드 3주차 히스토리 API 일정과 맞춤

**세부 항목**

- 히스토리 목록 화면 제작
- 채팅방별 제목 표시
- 최근 대화순 정렬 UI 반영
- GET `/api/history/{deviceId}` 연동
- GET `/api/history/{deviceId}/{chatRoomId}` 연동
- DELETE `/api/history/{chatRoomId}` 연동
- 삭제 확인 모달 구현
- 히스토리 없을 때 빈 상태 화면 구성

**완료 기준**

- 목록/상세/삭제 흐름 정상 동작
- 삭제 후 화면 즉시 반영
- 백엔드 응답 포맷과 프론트 데이터 구조 일치 확인

—

## 4주차 - FAQ 화면 구현 + 카테고리별 조회 기능

**목표:** FAQ 화면 완성 및 질문 선택 후 채팅 연결

**백엔드 연계 포인트:** 백엔드 4주차 FAQ API 일정과 맞춤

**세부 항목**

- FAQ 메인 화면 UI 제작
- 카테고리 탭 구성
- GET `/api/faq?category={카테고리}` 연동
- 카테고리별 목록 출력
- FAQ 질문 클릭 시 채팅 입력창으로 자동 연결
- 전체 FAQ 보기 기능 구현
- 자주 묻는 질문 카드형 UI 정리

**완료 기준**

- 카테고리 필터 동작 확인
- FAQ 선택 → 채팅 화면 이동 또는 질문 자동 입력 확인
- 초기 FAQ 데이터 노출 정상 확인

—

## 5주차 - 알림 키워드 등록/설정 화면 구현

**목표:** 알림 키워드 CRUD 및 알림 설정 화면 완성

**백엔드 연계 포인트:** 백엔드 5주차 알림 API, FCM 연동 시점과 맞춤

**세부 항목**

- 알림 설정 페이지 제작
- 키워드 등록 입력창 및 목록 UI 구현
- POST `/api/notification/keyword` 연동
- GET `/api/notification/keyword/{deviceId}` 연동
- DELETE `/api/notification/keyword/{keywordId}` 연동
- PUT `/api/notification/setting/{deviceId}` 연동
- 중복 등록, 글자 수 제한, 공백 입력 예외 메시지 처리
- 푸시 알림 권한 요청 화면 구성

**완료 기준**

- 키워드 등록/조회/삭제 정상 동작
- 알림 ON/OFF 설정 반영 확인
- 예외 상황 메시지 출력 확인

—

## 6주차 - UI 정리 + 배포 환경 대응

**목표:** 실서버 환경 대응 및 전체 화면 품질 개선

**백엔드 연계 포인트:** 백엔드 6주차 서버 배포 완료 후 실서버 주소 연동

**세부 항목**

- 개발 서버 주소 → 운영 서버 주소 변경 가능 구조 정리
- 환경변수 분리
- API base URL 설정
- 반응형 UI 점검
- 로딩 스피너, 토스트 메시지, 에러 페이지 정리
- 네트워크 오류/서버 오류 공통 처리
- 앱 아이콘, 스플래시, 기본 브랜딩 정리
- 최종 시연용 화면 다듬기

**완료 기준**

- 실서버 API 기준 정상 접속 확인
- 주요 화면 UI 정리 완료
- 공통 예외 처리 구조 정리 완료

—

## 7주차 - 통합 테스트 1차 + 버그 수정

**목표:** 전체 사용자 시나리오 테스트 및 수정

**세부 항목**

- 채팅 → 히스토리 저장 → 히스토리 조회 흐름 점검
- FAQ 선택 → 채팅 연결 흐름 점검
- 알림 키워드 등록 → 공지 발생 → 알림 수신 흐름 점검
- 서버 다운/응답 지연 시 예외 처리 점검
- 빈 데이터, 잘못된 요청, 삭제 후 재조회 등 경계 케이스 테스트
- UI 깨짐, 상태 반영 지연, 중복 클릭 문제 수정

**완료 기준**

- 주요 사용자 시나리오 테스트 통과
- 발견 버그 수정 완료
- 백엔드팀과 연동 이슈 정리 완료

—

## 8주차 - 통합 테스트 2차 + 시연 준비

**목표:** 최종 점검 및 발표/시연 준비 완료

**세부 항목**

- 실제 발표 흐름 기준 전체 기능 재점검
- 시연용 계정/데이터/키워드 사전 세팅
- 질문 예시 목록 준비
- 오류 발생 시 대체 시나리오 준비
- 발표용 스크린샷 및 화면 녹화 준비
- UI 최종 마감 및 문구 통일

**완료 기준**

- 전체 기능 안정 동작 확인
- 시연 시나리오 완성
- 발표 직전 점검 항목 정리 완료
