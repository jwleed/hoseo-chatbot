# 통합 개발 일정표

## 호서대학교 스마트 캠퍼스 도우미 (Hoseo-LENS)

**개발 인원:** 총 5명 (AI 1명, 백엔드 2명, 프론트엔드 2명)

**해당 기간:** 프로젝트 8주차 ~ 15주차 (후반부 8주 집중 개발)

---

### 1. 역할 분담

| **분야** | **담당자** | **담당 영역** |
| --- | --- | --- |
| **AI** | 조준희 | 라우터 파인튜닝, 데이터 자동화(크롤러) 및 벡터 DB 파이프라인, 프롬프트 엔지니어링, 상향식 테스트 총괄 |
| **백엔드** | 이재화 | 채팅 API, 히스토리 API, 서버 배포 |
| **백엔드** | 구자준 | FAQ API, 알림 서비스, FCM 연동, DB 설계 |
| **프론트엔드** | 프론트 1 | 채팅 화면, 히스토리 화면, API 연동 |
| **프론트엔드** | 프론트 2 | FAQ 화면, 알림/설정 화면, UI 공통 컴포넌트 |

---

### 2. 개발 우선순위

| **우선순위** | **기능** | **근거** |
| --- | --- | --- |
| **1** | 채팅 화면 및 API + AI 실연동 | 핵심 기능, 모든 기능의 기반 (SSE 스트리밍) |
| **2** | 데이터 자동화 및 벡터 DB 구축 | 최신 학사 정보 유지를 위한 필수 기능 |
| **3** | DB 환경 + 히스토리 화면/API | SRS 필수 기능 (연속적인 대화 맥락 유지) |
| **4** | FAQ 화면/API | SRS 중요 기능 (단순 반복 질의 처리) |
| **5** | 알림 키워드 설정 + FCM 연동 | SRS 중요 기능 (사용자 맞춤형 정보 제공) |
| **6** | 검색 고도화 및 프롬프트 튜닝 | 환각(Hallucination) 방지 및 품질 확보 핵심 |
| **7** | 클라우드 서버 배포 및 배포 환경 최적화 | 통합 테스트 및 운영 환경 구축을 위해 선행 완료 |

---

### 3. 주차별 세부 개발 일정 (8주차 ~ 15주차)

### 8주차 - 채팅 기본 연동 및 데이터 자동화 파이프라인

**목표:** FastAPI 연동 및 실시간 응답 환경 구축 / 프론트 UI 초안 / 자동 크롤링 모듈 구축

| **파트** | **세부 구현 항목** | **완료 기준** |
| --- | --- | --- |
| **AI** | • 공지사항/학칙 스크래핑(Scrapy/Selenium) 모듈 구현
• 신규 데이터 증분 추출 로직 작성 및 주기 실행 설정 | • 지정 시간 자동 크롤링 동작 확인
• 신규 게시글 증분 추출 확인 |
| **백엔드** | • AI 파트 FastAPI 서버 IP/포트 `rag.server.url` 설정
• Postman 실제 SSE chunk 수신 확인
• `sources` 필드(doc_id, title, url) 수신 확인 | • IP 설정 완료
• Postman SSE 및 sources 정상 수신 확인 |
| **프론트** | • 프로젝트 생성, 폴더/라우팅 구조 설계
• 공통 컬러, 폰트, 버튼, 입력창 스타일 정의
• 채팅 메인 화면 UI 및 로딩/빈 화면 제작
• 백엔드 채팅 API 명세 및 응답 구조 파악 | • 앱/웹 기본 실행 확인
• 채팅 화면 UI 1차 완성
• 공통 컴포넌트 구조 정리 |

**목표:** 전체 DB/테이블 생성 / SSE 스트리밍 UI 렌더링 / 하이브리드 검색 최적화

| **파트** | **세부 구현 항목** | **완료 기준** |
| --- | --- | --- |
| **AI** | • 크롤링 데이터 임베딩 파이프라인 및 Milvus 증분 업데이트
• 키워드(BM25) 및 의미(Dense) 검색 비율 최적화
• 환각 방지 시스템 프롬프트(가드레일) 최적화 | • 신규 데이터 벡터 DB 삽입 및 조회 확인

• `sources` 필드 표시 UI 구현

• 로딩/에러 UI 동작 확인 |
